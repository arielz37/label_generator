# 标签生成器安装教程（EXE 运行版）

本文档用于在现场电脑上部署已经打包好的 `LabelGenerator.exe`。  
如果使用的是 EXE 版本，现场电脑不需要安装 Python，也不需要执行 `pip install -r requirements.txt`。

## 1. 安装清单

目标电脑需要安装：

1. Microsoft Visual C++ Redistributable
2. Microsoft ODBC Driver 17 for SQL Server
3. BarTender
4. 标签打印机驱动

建议安装顺序：

```text
VC++ Redistributable
-> ODBC Driver 17
-> 标签打印机驱动
-> BarTender
-> 配置 ERP / BarTender 环境变量
-> 运行 LabelGenerator.exe
```

## 2. 安装 VC++ 运行库

ODBC Driver 17 可能需要 VC++ 运行库。  
64 位系统下载：

```text
https://aka.ms/vs/17/release/vc_redist.x64.exe
```

安装完成后建议重启电脑。

## 3. 安装 ODBC Driver 17 for SQL Server

下载页面：

```text
https://learn.microsoft.com/en-us/sql/connect/odbc/download-odbc-driver-for-sql-server
```

选择：

```text
Microsoft ODBC Driver 17 for SQL Server
```

如果安装时提示缺少 VC++，先安装上一节的 VC++ 运行库。

## 4. 安装标签打印机驱动

必须安装现场实际使用的标签打印机驱动。  
BarTender 的预览和打印都会依赖打印机驱动、纸张尺寸和页面设置。

安装后，在 Windows 打印机列表中确认打印机名称，例如：

```text
TSC TTP-244 Pro
ZDesigner ZD230-203dpi ZPL
```

后续配置 `BARTENDER_PRINTER` 时要使用这个名称。

## 5. 安装 BarTender

安装 BarTender 后，确认电脑上存在：

```text
bartend.exe
```

程序会自动扫描常见路径：

```text
C:\Program Files\Seagull\BarTender Suite\bartend.exe
C:\Program Files (x86)\Seagull\BarTender Suite\bartend.exe
C:\Program Files\Seagull\BarTender*\bartend.exe
C:\Program Files (x86)\Seagull\BarTender*\bartend.exe
```

如果 BarTender 安装在特殊路径，需要配置环境变量：

```powershell
setx BARTENDER_EXE "实际路径\bartend.exe"
```

程序界面里有 `打印机` 下拉框，生成预览和打印都会使用这里选择的打印机。首次运行时请确认选择的是现场实际标签打印机。

如果希望程序打开时默认选中某台打印机，可以配置环境变量：

```powershell
setx BARTENDER_PRINTER "实际标签打印机名称"
```

打印机名称要和 Windows 打印机列表里显示的名称一致。配置后需要重新打开程序。也可以不配置环境变量，直接在界面下拉框里选择。

## 6. 准备程序目录

将发布包复制到目标电脑，例如：

```text
C:\LabelGenerator
```

建议不要放到：

```text
C:\Program Files
```

因为程序运行时会写入：

```text
runtime_data/
final_labels/
logs/
```

如果放在 `Program Files`，可能遇到写入权限问题。

发布目录建议包含：

```text
LabelGenerator.exe
Templates/   （空目录，现场导入模板后再放入）
template_mapping.xlsx
docs/
```

运行时目录会自动生成：

```text
runtime_data/
final_labels/
logs/
```

如果是在开发电脑上重新打包 EXE，使用：

```powershell
pip install -r requirements.txt
pip install -r requirements-build.txt
.\build_exe.ps1
```

打包结果在：

```text
dist\LabelGenerator\
```

发布到现场电脑时，请复制整个 `dist\LabelGenerator\` 文件夹，不要只复制单个 EXE。
打包脚本只负责复制配置文件和创建空的 `Templates/` 目录，不会把开发电脑上的 `.btw` 模板打包进去，也不会自动修改模板数据库连接。首次部署到现场电脑后，请把现场模板放入 `Templates/` 或其他目录，再在 EXE 界面点击 `工具 -> 模板数据库设置`，选择要处理的模板目录批量绑定数据库连接。

## 7. 配置 ERP 连接参数

程序从环境变量读取 ERP 参数。

永久配置：

```powershell
setx ERP_SERVER "服务器地址或IP"
setx ERP_DATABASE "DB_BD01"
setx ERP_USER "ERP账号"
setx ERP_PASSWORD "ERP密码"
setx ERP_DRIVER "ODBC Driver 17 for SQL Server"
```

使用 `setx` 后，需要重新打开程序才会生效。

## 8. 运行程序

双击运行：

```text
LabelGenerator.exe
```

打开后：

1. 输入 MO 号，然后按 Enter 或离开输入框，程序会自动识别外标/内标模板并填入路径
2. 确认 `打印机` 下拉框选择的是现场标签打印机
3. 如果需要临时改用其他模板，可以点击 `修改外标` / `修改内标` 选择新的 `.btw`，也可以直接手动编辑模板路径
4. 如果需要调整字段绑定，可以点击 `打开外标模板` / `打开内标模板`，程序会按 UI 指定模板优先、否则按 MO 自动查找模板并用 BarTender 打开
5. 点击 `生成预览`
6. 查看外标/内标预览
7. 确认后点击打印按钮

注意：如果在生成预览后修改了打印机或手动指定模板，程序会要求重新生成预览后再打印，避免预览和实际打印使用不同配置。

如果是第一次在这台电脑上导入模板，可以点击界面上的 `工具 -> 模板数据库设置`，手动选择要处理的模板目录，批量给该目录下的 `.btw` 模板添加 `runtime_data/current_label.csv` 数据库连接。

如果导入了新的模板目录，需要把新模板追加到 `template_mapping.xlsx`，可以点击 `工具 -> 更新模板映射`，选择要扫描的目录。程序只会把未出现过的模板路径追加到表格底部，不会修改原来已经存在的数据。

如果现场模板不放在默认的 `Templates/` 目录，`template_mapping.xlsx` 或 `bom料号名汇总.txt` 放在了其他位置，可以点击 `工具 -> 更改参数和路径` 修改：

```text
模板搜索目录
BOM料号名汇总
模板映射表
```

保存后配置会写入程序目录下的 `app_settings.json`，并立即影响后续模板查找、模板映射更新和BOM内包材名称匹配。

## 9. BarTender 模板要求

模板最终必须在 BarTender 中绑定：

```text
runtime_data/current_label.csv
```

CSV 第一行是字段名。当前字段：

```text
MO_NO
ORDER_NO
CUS_NO
SUP_PRD_NO
MRP_NO
MFG_QTY
MFG_DATE
MFG_DATE_YYMMDD
MFG_DATE_YYYYMMDD
EXP_DATE
EXP_DATE_YYMMDD
EXP_DATE_YYYYMMDD
LOT_NO
HAS_INNER_LABEL
QTY
LABEL_COUNT
LABEL_INDEX
PACKAGE_PRD_NO
PACKAGE_NAME
```

常用绑定：

```text
制令单号 -> MO_NO
订单号   -> ORDER_NO
LOT NO   -> LOT_NO
客户料号 -> SUP_PRD_NO
产品编号 -> MRP_NO
制造数量 -> MFG_QTY
生产日期 -> MFG_DATE
生产日期YYMMDD   -> MFG_DATE_YYMMDD
生产日期YYYYMMDD -> MFG_DATE_YYYYMMDD
有效期   -> EXP_DATE
有效期YYMMDD     -> EXP_DATE_YYMMDD
有效期YYYYMMDD   -> EXP_DATE_YYYYMMDD
数量     -> QTY
序号     -> LABEL_INDEX
包材编号 -> PACKAGE_PRD_NO
包材名称 -> PACKAGE_NAME
```

程序采用：

```text
CSV 一行 = 一张标签
```

如果制造数量不是整箱/整袋，程序会先生成整标标签，再追加一张零数标签。零数标签这一行的 `QTY` 是余数数量。

例如制造数量 `1000`，每箱数量 `600`：

```text
第 1 行：QTY=600，LABEL_INDEX=1，LABEL_COUNT=2
第 2 行：QTY=400，LABEL_INDEX=2，LABEL_COUNT=2
```

外标触发零数标签时，界面会用红字提醒“请人工额外打印零数箱标签”。

预览生成时，程序会通过 BarTender XMLScript 临时强制每条记录打印份数为 1。模板最好也保持为 1。

注意：预览图片的数量是“打印预览页数”，不是绝对等于标签张数。如果电脑没有匹配正确的标签打印机驱动，BarTender 可能按错误纸张尺寸排版，出现一页多标或分页异常。此时应检查界面里选择的 `打印机` 是否正确；`BARTENDER_PRINTER` 只作为默认选中值。

## 10. 批量设置模板数据库连接

这一节用于模板维护电脑。  
如果只是运行已经打包好的 `LabelGenerator.exe`，通常不需要执行本节。

EXE 版本可以直接在界面点击 `工具 -> 模板数据库设置`，然后选择要处理的模板目录；不需要现场电脑为了这一步单独安装 Python。

如果要用命令行执行下面的脚本，维护电脑才需要 Python 环境、pywin32、BarTender，并且 BarTender 版本支持 XMLScript。

当模板很多时，可以用脚本批量给 `.btw` 模板添加 CSV 数据库连接：

```powershell
python docs\batch_set_bartender_database.py --limit 3
```

这一步是 dry-run，不会保存模板。确认报告无误后，先小范围保存测试：

```powershell
python docs\batch_set_bartender_database.py --limit 3 --save --visible
```

确认 BarTender 里能看到数据库字段后，再全量处理：

```powershell
python docs\batch_set_bartender_database.py --save
```

脚本会把模板绑定到：

```text
runtime_data/current_label.csv
```

如果模板没有数据库连接，脚本会通过 BarTender XMLScript 自动新增。新电脑重新导入模板时，建议把模板当作全新模板批量设置数据库连接。

界面按钮默认使用 XMLScript-only 方式处理模板，不再用 COM 预先打开模板检查连接数量，目的是减少旧电脑/旧 BarTender 上的兼容弹窗。

如果运行时出现：

```text
文件由新版本 BarTender 创建，无法加载
文件被更高版本 BarTender 修改，将无法通过旧版本 BarTender 打开
```

这属于模板版本不兼容，程序无法绕过。需要用现场电脑安装相同或更高版本 BarTender，或者在新版本 BarTender 中把模板另存为现场 BarTender 能打开的版本。

如果出现“打印机自动改为默认”之类提示，请先确认界面 `打印机` 下拉框选择了现场标签打印机。数据库设置脚本会把该打印机写入 XMLScript，尽量避免保存模板时被 BarTender 改成系统默认打印机。

使用 `--save` 时，原模板会先备份到：

```text
template_backups/
```

处理报告生成在：

```text
logs/batch_set_bartender_database_report.csv
```

注意：这个脚本只负责“数据库设置”。每个文本框、条码对象具体绑定哪个字段，仍然需要在 BarTender 里人工调整。

## 11. 输出目录说明

运行时生成：

```text
runtime_data/current_label.csv
final_labels/
logs/
```

`runtime_data/current_label.csv` 会被反复覆盖：

```text
先写外标数据（整标行 + 可能存在的零数行）
生成外标预览/打印
再写内标数据（整标行 + 可能存在的零数行）
生成内标预览/打印
```

这是正常行为。

预览图片会生成到：

```text
final_labels/previews/
```

预览图片属于临时缓存。程序每次重新生成同一个 MO 的预览时，会先清理这个 MO 的旧 PNG；同时只保留最近一批预览任务和 XML 脚本，避免长期占用磁盘空间。

打印任务记录会生成到：

```text
final_labels/label_job_*.json
```

桌面程序运行日志、工单信息记录和模板数据库设置报告会生成到：

```text
logs/
```

程序会自动清理 `logs/` 中超过 7 天的日志和报告文件。

## 12. 常见问题

### 1. 找不到 `bartend.exe`

如果报：

```text
FileNotFoundError: bartend.exe
```

说明程序没有找到 BarTender。

解决：

```powershell
setx BARTENDER_EXE "实际路径\bartend.exe"
```

重新打开程序后再运行。

### 2. 找不到 ODBC Driver 17

确认已经安装：

```text
Microsoft ODBC Driver 17 for SQL Server
```

并确认环境变量：

```powershell
setx ERP_DRIVER "ODBC Driver 17 for SQL Server"
```

### 3. ERP 参数为空或连接失败

检查这些环境变量是否配置：

```text
ERP_SERVER
ERP_DATABASE
ERP_USER
ERP_PASSWORD
ERP_DRIVER
```

使用 `setx` 配置后，一定要重新打开程序。

### 4. 预览数量不对

优先检查程序界面里的 `打印机` 是否选择了正确标签打印机。

如果希望固定默认值，也可以配置：

```text
BARTENDER_PRINTER = 实际标签打印机名称
```

预览图片数量是打印预览页数。正常生产环境应使用正确标签打印机驱动；如果默认打印机不匹配，BarTender 可能会把多个标签排到同一页，导致预览页数少于 CSV 标签行数。

如果实际打印张数被放大，再检查 BarTender 模板：

```text
每条记录打印份数 = 1
```

### 5. 中文路径或中文文件名乱码

程序内部使用 UTF-8 保存 CSV。BarTender 模板路径可以包含中文，但如果旧电脑环境异常，建议项目目录尽量使用英文路径，例如：

```text
C:\LabelGenerator
```
