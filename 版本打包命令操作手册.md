# 版本打包命令操作手册

## 一页速查

先在项目目录打开 PowerShell，然后根据需要执行一条命令：

| 要做什么 | 执行命令 | 版本变化 |
| --- | --- | --- |
| 普通打包，不升级版本 | `.\build_exe.ps1` | `v1.2.2 → v1.2.2` |
| 修复 Bug，升级小版本 | `.\build_exe.ps1 -Bump patch` | `v1.2.2 → v1.2.3` |
| 增加新功能，升级功能版本 | `.\build_exe.ps1 -Bump minor` | `v1.2.2 → v1.3.0` |
| 大规模改版，升级大版本 | `.\build_exe.ps1 -Bump major` | `v1.2.2 → v2.0.0` |

项目目录：

```powershell
cd C:\Users\58372\Documents\label_generator
```

最常用的两条命令：

```powershell
# 测试或重新打包，不升级版本
.\build_exe.ps1

# 修复完成后正式发布，升级最后一位版本号
.\build_exe.ps1 -Bump patch
```

> 注意：带 `-Bump` 的命令只执行一次。已经升版后如果需要重新打包，请使用 `.\build_exe.ps1`，否则版本号会再次增加。

## 一、打开命令行

进入项目目录：

```powershell
cd C:\Users\58372\Documents\label_generator
```

以下命令均在 PowerShell 中执行。

## 二、打包但不升级版本

如果只是测试打包、重新打包，或者希望继续使用当前版本号，执行：

```powershell
.\build_exe.ps1
```

例如当前版本是 `v1.2.2`，打包完成后仍然是 `v1.2.2`。

这是平时最常用的打包命令。

## 三、打包并升级修复版本

如果本次主要是修复 Bug、小范围调整，执行：

```powershell
.\build_exe.ps1 -Bump patch
```

版本变化示例：

```text
v1.2.2 → v1.2.3
```

`patch` 表示升级最后一位版本号。

## 四、打包并升级功能版本

如果本次增加了新功能，并且不影响原来的使用方式，执行：

```powershell
.\build_exe.ps1 -Bump minor
```

版本变化示例：

```text
v1.2.2 → v1.3.0
```

`minor` 表示升级中间一位版本号，同时把最后一位重置为 `0`。

## 五、打包并升级大版本

如果本次进行了大规模改版，或者新版本与旧版本不兼容，执行：

```powershell
.\build_exe.ps1 -Bump major
```

版本变化示例：

```text
v1.2.2 → v2.0.0
```

`major` 表示升级第一位版本号，同时把后两位重置为 `0`。

## 六、常用命令对照

| 使用场景 | 命令 | 版本变化示例 |
| --- | --- | --- |
| 测试或重新打包，不升级版本 | `.\build_exe.ps1` | `v1.2.2 → v1.2.2` |
| 修复 Bug 后正式发布 | `.\build_exe.ps1 -Bump patch` | `v1.2.2 → v1.2.3` |
| 增加新功能后正式发布 | `.\build_exe.ps1 -Bump minor` | `v1.2.2 → v1.3.0` |
| 大规模或不兼容改版 | `.\build_exe.ps1 -Bump major` | `v1.2.2 → v2.0.0` |

## 七、重要注意事项

同一个版本只执行一次带 `-Bump` 的命令。

例如第一次执行：

```powershell
.\build_exe.ps1 -Bump patch
```

版本会从 `v1.2.2` 升级为 `v1.2.3`。

如果打包失败，修复问题后需要重新打包 `v1.2.3`，应执行：

```powershell
.\build_exe.ps1
```

不要再次执行 `-Bump patch`，否则版本会继续升级为 `v1.2.4`。

## 八、查看当前版本

当前版本统一记录在项目根目录的 `version.py`：

```python
__version__ = "1.2.2"
```

一般不需要手工修改该文件。执行带 `-Bump` 的打包命令时，脚本会自动修改版本号；程序右下角也会自动显示相同版本。

## 九、打包输出位置

打包成功后，程序位于：

```text
dist\LabelGenerator\
```

发布时应复制整个 `dist\LabelGenerator\` 文件夹，不要只复制其中的 EXE 文件。
