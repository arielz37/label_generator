字段变量说明
============

一、BarTender 模板绑定常用字段
-----------------------------

这些字段会出现在 runtime_data/current_label.csv 里，也是 BarTender 模板数据库字段绑定时最常用的字段。

| 字段名 | 含义 | 来源/用途 | 示例 |
|---|---|---|---|
| MO_NO | 制令单号 / MO号 | ERP 工单号 | MO26061617 |
| SO_NO | SO号 / 销售单号 | ERP 工单对应 SO_NO，通常带 SO 前缀 | SO26060001 |
| CUS_OS_NO | ERP客户订单字段 | ERP 里定位到的客户订单号来源字段 | 7226003957 |
| ORDER_NO | 客户订单号 | 给模板使用的客户订单号，目前等于 CUS_OS_NO | 7226003957 |
| CUS_NO | 客户编号 | ERP 工单对应客户 | SHHXDZ001 |
| SUP_PRD_NO | 客户料号 | 客户侧产品料号，常用于标签上的客户料号文本 | 59-393002-04 |
| MRP_NO | 公司内部产品编号 / 成品料号 | 公司内部产品编码，常用于产品编号文本 | TDLAS0010_SHHXDZ |
| MFG_QTY | 制造数量 | ERP 工单制造总数量；注意它不是单张标签数量 | 8400 |
| MFG_DATE | 生产日期 | 来源于 ERP 工单生产日期 | 2026-06-16 |
| MFG_DATE_YYMMDD | 特殊格式生产日期 | 生产日期的 2位年 + 2位月 + 2位日格式 | 260616 |
| MFG_DATE_YYYYMMDD | 特殊格式生产日期 | 生产日期的 4位年 + 2位月 + 2位日格式，中间无横线 | 20260616 |
| MFG_DATE_DD_MM_YYYY_DOT | 点分隔生产日期 | 日.月.年格式 | 16.06.2026 |
| MFG_DATE_YYYY_MM_DD_DOT | 点分隔生产日期 | 年.月.日格式 | 2026.06.16 |
| MFG_DATE_YY_MM_DD_DOT | 点分隔生产日期 | 2位年.月.日格式 | 26.06.16 |
| EXP_DATE | 有效期 / 过期日期 | 由生产日期 + 保质期 - 1天计算得到；UI 手动选择保质期时以 UI 为准 | 2027-06-15 |
| EXP_DATE_YYMMDD | 特殊格式有效期 | 有效期的 2位年 + 2位月 + 2位日格式 | 270615 |
| EXP_DATE_YYYYMMDD | 特殊格式有效期 | 有效期的 4位年 + 2位月 + 2位日格式，中间无横线 | 20270615 |
| EXP_DATE_DD_MM_YYYY_DOT | 点分隔有效期 | 日.月.年格式 | 15.06.2027 |
| EXP_DATE_YYYY_MM_DD_DOT | 点分隔有效期 | 年.月.日格式 | 2027.06.15 |
| EXP_DATE_YY_MM_DD_DOT | 点分隔有效期 | 2位年.月.日格式 | 27.06.15 |
| LOT_NO | LOT NO | 由生产日期YYMMDD、MO号、固定后缀SZBD用 - 拼接 | 260608-MO26060908-SZBD |
| HAS_INNER_LABEL | 是否有内标 | Y 表示有内标，N 表示无内标 | Y |
| QTY | 当前这张标签显示的数量 | 外标时表示外标数量，内标时表示内标数量；零数标签时表示余数数量 | 1200 / 600 / 400 |
| LABEL_COUNT | 当前标签类型的总张数 | 当前外标或内标一共需要生成/打印多少张 | 7 |
| LABEL_INDEX | 当前标签序号 | 从 1 开始递增，可用于标签序号显示 | 1 |
| PACKAGE_PRD_NO | 当前标签对应的包材编号 | 外标通常是纸箱/外包装料号，内标通常是袋/罐/桶等内包装料号 | BP350X350X315B |
| PACKAGE_NAME | 当前标签对应的包材名称 | 当前外标或内标使用的包材名称 | 纸箱 / 双层防潮袋 |


二、程序内部计算字段
-------------------

这些字段主要在程序内部、工单信息、日志中使用。最终写给 BarTender 的 current_label.csv 会统一折算成 QTY / LABEL_COUNT / PACKAGE_PRD_NO / PACKAGE_NAME。

| 字段名 | 含义 | 来源/用途 | 示例 |
|---|---|---|---|
| INNER_QTY | 内标每张标签对应的产品数量 | 来源于 ERP BOM 包装用量基数 | 600 |
| OUTER_QTY | 外标每张标签对应的产品数量 | 来源于 ERP BOM 包装用量基数 | 1200 |
| INNER_LABEL_COUNT | 内标整标张数 | 制造数量 / INNER_QTY 向下取整；余数会额外生成零数标签 | 14 |
| OUTER_LABEL_COUNT | 外标整标张数 | 制造数量 / OUTER_QTY 向下取整；余数会额外生成零数标签 | 7 |
| INNER_PACKAGE_PRD_NO | 内标包材编号 | ERP BOM 中识别出的内包装料号 | MML/LEA580X470KX9/6 |
| INNER_PACKAGE_NAME | 内标包材名称 | ERP BOM 中识别出的内包装名称 | 双层防潮袋 |
| OUTER_PACKAGE_PRD_NO | 外标包材编号 | ERP BOM 中识别出的外包装料号 | BP350X350X315B |
| OUTER_PACKAGE_NAME | 外标包材名称 | ERP BOM 中识别出的外包装名称 | 纸箱 |
| PACKAGE_SOURCE | 包装数量和包材信息来源 | 当前主要来源为 BOM | BOM |
| INNER_PACKAGE_MISSING | 内标包装信息是否缺失 | 如果 ERP/BOM 没有查到内标包装数量，UI 会提醒内标信息缺失 | Y / 空 |


三、模板绑定建议
---------------

| 标签内容 | 建议绑定字段 | 说明 |
|---|---|---|
| 制令单号 / MO号 | MO_NO | 需要显示工单号时使用 |
| SO号 / 销售单号 | SO_NO | 需要显示 ERP SO 号时使用 |
| ERP客户订单字段 | CUS_OS_NO | 需要直接绑定 ERP 原字段时使用 |
| 客户订单号 | ORDER_NO | 需要显示客户侧订单号时使用 |
| LOT NO | LOT_NO | 需要显示类似 260608-MO26060908-SZBD 的批号时使用 |
| 客户编号 | CUS_NO | 需要显示客户编号时使用 |
| 客户料号 | SUP_PRD_NO | 常用字段 |
| 产品编号 / 成品料号 | MRP_NO | 常用字段 |
| 制造数量 | MFG_QTY | 工单制造总数量 |
| 生产日期 | MFG_DATE | 日期格式由程序生成 |
| 生产日期 YYMMDD | MFG_DATE_YYMMDD | 例如 260616 |
| 生产日期 YYYYMMDD | MFG_DATE_YYYYMMDD | 例如 20260616 |
| 生产日期 DD.MM.YYYY | MFG_DATE_DD_MM_YYYY_DOT | 例如 16.06.2026 |
| 生产日期 YYYY.MM.DD | MFG_DATE_YYYY_MM_DD_DOT | 例如 2026.06.16 |
| 生产日期 YY.MM.DD | MFG_DATE_YY_MM_DD_DOT | 例如 26.06.16 |
| 有效期 | EXP_DATE | 日期格式由程序生成 |
| 有效期 YYMMDD | EXP_DATE_YYMMDD | 例如 270615 |
| 有效期 YYYYMMDD | EXP_DATE_YYYYMMDD | 例如 20270615 |
| 有效期 DD.MM.YYYY | EXP_DATE_DD_MM_YYYY_DOT | 例如 15.06.2027 |
| 有效期 YYYY.MM.DD | EXP_DATE_YYYY_MM_DD_DOT | 例如 2027.06.15 |
| 有效期 YY.MM.DD | EXP_DATE_YY_MM_DD_DOT | 例如 27.06.15 |
| 数量 | QTY | 内标外标共用模板时也绑定这个字段 |
| 总张数 | LABEL_COUNT | 需要显示总张数时使用 |
| 当前序号 | LABEL_INDEX | 需要显示第几张时使用 |
| 包材编号 | PACKAGE_PRD_NO | 需要显示包材料号时使用 |
| 包材名称 | PACKAGE_NAME | 需要显示包材名称时使用 |


四、注意事项
-----------

| 注意点 | 说明 |
|---|---|
| BarTender 模板一般只需要绑定第一部分字段 | 第二部分是程序内部字段，通常不用在模板里绑定 |
| QTY 会自动变化 | 外标生成时是外标数量，内标生成时是内标数量 |
| 内外标共用同一个 .btw 模板 | 仍然绑定 QTY，不需要分别绑定 INNER_QTY / OUTER_QTY |
| CSV 一行代表一张标签 | 程序通过生成多行 CSV 控制打印/预览张数 |
| 零数标签 | 如果制造数量不能整除每标数量，程序会追加一行零数标签，QTY 为余数 |
