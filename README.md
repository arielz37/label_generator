# Label Generator

This project generates BarTender label preview images and print jobs from ERP manufacturing orders.

Current scope:

1. Query ERP data by `MO_NO`.
2. Generate `runtime_data/current_label.csv`.
3. Calculate expiration date, label quantities, and label counts from ERP BOM data.
4. Locate the required `.btw` template from `template_mapping.xlsx`, unless the desktop UI manually specifies an outer/inner template. The UI can also open the resolved outer/inner template directly in BarTender for field binding.
5. Decide whether an inner label exists from both ERP/BOM inner package quantity and an inner/combined template mapping.
6. Generate preview images and optionally print through BarTender CLI.

## Requirements

Python 3.8 is the target runtime.

Install Python dependencies:

```bash
pip install -r requirements.txt
```

ERP access also requires Microsoft ODBC Driver 17 for SQL Server to be installed on the machine.

Build-only dependencies:

```bash
pip install -r requirements-build.txt
```

Set ERP connection environment variables before running ERP queries:

```bash
export ERP_SERVER="your_sql_server"
export ERP_DATABASE="DB_BD01"
export ERP_USER="your_user"
export ERP_PASSWORD="your_password"
```

Optional:

```bash
export ERP_DRIVER="ODBC Driver 17 for SQL Server"
export BARTENDER_PRINTER="your_label_printer_name"
```

The desktop UI has a printer selector. `BARTENDER_PRINTER` is only the default selected printer; operators can choose the actual label printer in the UI before generating previews and printing.

## Main Files

- `desktop_app.py`  
  Desktop GUI entry. Enter an MO number, generate label previews, then print.

- `label_service.py`  
  Orchestrates ERP lookup, runtime CSV generation, template lookup, preview generation, and printing.

- `erp_runtime_csv.py`  
  Queries ERP `MF_MO` data and generates runtime CSV.

- `template_mapping_lookup.py`  
  Reads `template_mapping.xlsx` and locates the `.btw` template.

- `generate_template_mapping.py`  
  Scans `Templates/` and generates an initial template mapping workbook.
  The desktop app also exposes this under `工具 -> 更新模板映射`; it scans a selected directory and only appends new template paths to `template_mapping.xlsx`.

- `bartender_label_runner.py`  
  Builds and optionally runs BarTender CLI print commands.

- `bartender_preview_runner.py`  
  Uses BarTender XMLScript to export print-preview images.

- `docs/batch_set_bartender_database.py`  
  Maintenance tool for batch-adding the CSV database connection in newly imported `.btw` templates.
  The desktop app also exposes this under `工具 -> 模板数据库设置`.

- `app_settings.json`  
  Optional local settings file generated by `工具 -> 更改参数和路径`.
  It can override the template search directory, template mapping workbook, and BOM package-name text file path without changing code.

## Runtime CSV Fields

The current runtime CSV fields are:

```text
MO_NO
SO_NO
CUS_OS_NO
ORDER_NO
CUS_NO
SUP_PRD_NO
MRP_NO
MFG_QTY
MFG_DATE
MFG_DATE_YYMMDD
MFG_DATE_YYYYMMDD
EXP_DATE
EXP_DATE_YYMMDD
EXP_DATE_YYYYMMDD
LOT_NO
HAS_INNER_LABEL
QTY
LABEL_COUNT
LABEL_INDEX
PACKAGE_PRD_NO
PACKAGE_NAME
```

`SUP_PRD_NO` is the customer part number.  
`MRP_NO` is the product number.
`MFG_QTY` is the manufacturing order quantity.
`SO_NO` is the ERP sales order number, usually beginning with `SO`.
`CUS_OS_NO` is the ERP customer order number source field.
`ORDER_NO` is the customer purchase/order number; currently it uses `CUS_OS_NO`.
`LOT_NO` is generated as `MFG_DATE_YYMMDD-MO_NO-SZBD`.
`QTY` is the quantity shown on the current label type.
`LABEL_COUNT` is the total label count for that label type, including a final remainder label when one is generated.
`LABEL_INDEX` is one-based and makes the CSV one row per label.
`PACKAGE_PRD_NO` and `PACKAGE_NAME` describe the package material used for the current label type.

If the production quantity is not an exact multiple of the package quantity, the app writes normal full-package rows first and appends one remainder row at the end. For example, production quantity `1000` and outer package quantity `600` produces:

```text
row 1: QTY=600, LABEL_INDEX=1, LABEL_COUNT=2
row 2: QTY=400, LABEL_INDEX=2, LABEL_COUNT=2
```

Outer remainder labels are highlighted in the UI because the operator must manually handle the partial box label.

Preview generation forces BarTender copies per record to `1` through XMLScript; label count is controlled by CSV rows.
Preview images are print-preview pages, so one image may contain multiple labels when the template uses a multi-label page layout.

## Desktop App

Run:

```bash
python desktop_app.py
```

The app writes temporary data under:

```text
runtime_data/
final_labels/
logs/
```

These folders are generated output and should not be packaged as source data.
Preview PNG files under `final_labels/previews/` are temporary cache files. The app clears old previews for the same MO before regenerating and keeps only recent preview jobs.
Desktop operation logs, saved work-order info, and template database setup reports are written under `logs/`. Files older than 7 days are cleaned automatically.

## Build EXE

Build on a Windows machine with Python 3.8:

```powershell
pip install -r requirements.txt
pip install -r requirements-build.txt
.\build_exe.ps1
```

The output folder is:

```text
dist/LabelGenerator/
```

The build script copies/creates these external runtime files next to the EXE:

```text
Templates/ (empty directory; import site templates here later)
template_mapping.xlsx
bom料号名汇总.txt
docs/安装教程.md
```

`.btw` template files are not packaged into the EXE release folder.

Use the whole `dist/LabelGenerator/` folder as the release package.

## Mapping Workbooks

### `template_mapping.xlsx`

Template file mapping. Lookup key:

```text
客户编号 + 匹配编号 + 标签类型
```

Main output:

```text
模版相对路径
```

If only an `外标` mapping exists for a product, it is treated as no-inner-label.
If an `内标` or `内外标` mapping exists, it is treated as having an inner label.

## BarTender Notes

Python does not decide where fields appear on the label. BarTender templates must bind their objects to the runtime CSV field names.

Example:

```text
MFG_DATE -> production date text object
EXP_DATE -> expiration date text object
SUP_PRD_NO -> customer part number text object
```

### Batch Set Template Database

For template maintenance, you can batch add the BarTender CSV database connection to newly imported templates:

Normal EXE usage: click `工具 -> 模板数据库设置` in the desktop app, then select the template directory to process.

Developer command-line usage:

```powershell
python docs\batch_set_bartender_database.py --limit 3
python docs\batch_set_bartender_database.py --limit 3 --save --visible
python docs\batch_set_bartender_database.py --save
```

The tool binds templates to:

```text
runtime_data/current_label.csv
```

If a template has no database connection, the tool uses BarTender XMLScript to add one. When `--save` is used, original templates are backed up under `template_backups/`.
The report is written under `logs/`.

After this step, each template object still needs to be manually bound to the correct CSV field in BarTender.
